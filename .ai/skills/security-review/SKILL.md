# Skill: security-review

## Purpose

Check secrets, auth, authorization, injection, unsafe dependencies, risky data exposure.

## When to use

Automatically selected by orchestrator.

## Output

- Findings
- Recommended action
- Risks
- Files affected, if relevant

## Rules

- Be concise.
- Be evidence-based.
- Do not invent facts.
- Prefer durable knowledge updates when relevant.
