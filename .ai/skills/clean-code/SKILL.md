# Skill: clean-code

## Purpose

Ensure code is modular, readable, small, and separated by concern.

## When to use

Automatically selected by orchestrator.

## Output

- Findings
- Recommended action
- Risks
- Files affected, if relevant

## Rules

- Be concise.
- Be evidence-based.
- Do not invent facts.
- Prefer durable knowledge updates when relevant.
