# Context

This folder is the approved project brain.

Do not treat drafts as approved.

## Files

- PROJECT.md: identity, vision, scope, users
- PRD.md: features, user stories, acceptance criteria
- ARCHITECTURE.md: system design and data flow
- IMPLEMENTATION.md: technical implementation plan
- DECISIONS.md: approved decisions
- CHANGELOG.md: project changes
- manifest.json: version and approval status

## Rule

If implementation changes the core understanding of the project, update context.
