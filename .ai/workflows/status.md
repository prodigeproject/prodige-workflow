# Workflow: Status

1. Load memory bank files (activeContext, projectState, systemState).
2. Check git status (branch, uncommitted changes, last commit).
3. Read CHANGELOG.md for recent changes.
4. Check open threads and decisions pending approval.
5. Run verification suite (tests, lint, types, build).
6. Collect verification results and error counts.
7. Check cache freshness and last sync time.
8. Review active work items and in-progress features.
9. Calculate project health metrics (pass rate, coverage, issues).
10. Identify blockers and technical debt.
11. Check context/code drift from last sync.
12. Generate status summary with color-coded health indicators.
13. Highlight action items and recommendations.
14. Report findings in structured format.
15. Suggest next steps if issues found.
