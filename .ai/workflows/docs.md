# Workflow: Docs

1. Load context and memory bank.
2. Identify documentation scope (README, API, guides, decisions).
3. Read current documentation files.
4. Compare docs with current implementation.
5. Identify gaps and outdated sections.
6. Ask clarifying questions about audience and depth.
7. Draft documentation updates (surgical, not wholesale).
8. Include working code examples.
9. Verify all links and references.
10. Update CHANGELOG.md for user-facing changes.
11. Update DECISIONS.md if architectural choices are documented.
12. Run verification checks if applicable.
13. Self-check: beginner-friendly, examples work, quick start exists.
14. Request review if major documentation changes.
15. Update memory bank and handoff.
